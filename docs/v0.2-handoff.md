## 项目概述

taskwarrior-motion 是一个本地任务管理 Web UI，基于 taskwarrior 3.x CLI。

- **技术栈**：Vue 3 + Vite + TypeScript + Tailwind CSS v4（前端）| Rust + Axum（后端）
- **当前版本**：v0.1.0（已打 tag，release 分支已推送）
- **仓库**：https://github.com/GZ0759/taskwarrior-motion

## v0.1 已完成的功能

### 前端
- 双栏布局（左侧面板 + 右侧面板）
- 玻璃拟态 UI + Mesh 渐变背景
- 事项视图（待办列表 + 已完成折叠）
- 看板视图（5 列）
- 日历视图（月/周）
- 热力图（35 天，点击查看当日完成）
- 项目进度条（点击管理）
- 标签列表（点击管理）
- 任务卡片（两行布局：描述+日期 / 标签+优先级+计时+编辑）
- 6 个弹窗（编辑/完成/热力图/项目/标签/帮助）
- 暗色/亮色/系统主题三态切换
- 键盘快捷键（j/k/n/x/Enter/Escape/?）
- 时间追踪（开始/停止计时）
- Web Audio API 音效

### 后端
- Rust + Axum API 服务
- 视图专用接口（pending/completed/calendar/stats）
- 按日期查询已完成任务
- 任务状态操作（完成/取消完成/开始/停止/撤销）
- 请求日志（tracing）
- CORS 限制

## 后端 API 一览

| 方法 | 端点 | 用途 |
|------|------|------|
| GET | `/api/tasks` | 获取所有任务 |
| POST | `/api/tasks` | 创建任务 |
| PUT | `/api/tasks/:uuid` | 更新任务 |
| DELETE | `/api/tasks/:uuid` | 删除任务 |
| GET | `/api/tasks/pending` | 待办任务 |
| GET | `/api/tasks/completed?days=N` | 最近 N 天完成 |
| GET | `/api/tasks/completed?date=YYYY-MM-DD` | 指定日期完成 |
| GET | `/api/tasks/calendar` | 有截止日期的任务 |
| GET | `/api/stats` | 热力图 + 项目统计 |
| POST | `/api/tasks/:uuid/done` | 完成任务 |
| POST | `/api/tasks/:uuid/uncomplete` | 取消完成 |
| POST | `/api/tasks/:uuid/start` | 开始计时 |
| POST | `/api/tasks/:uuid/stop` | 停止计时 |
| POST | `/api/undo` | 撤销 |

### 数据模型

```typescript
interface Task {
  uuid: string
  description: string
  status: 'pending' | 'completed' | 'deleted'
  project?: string
  tags?: string[]
  priority?: 'H' | 'M' | 'L'
  due?: string           // YYYYMMDDTHHmmssZ
  wait?: string
  start?: string
  end?: string           // 完成时间
  entry: string
  urgency: number
  id: number
}
```

## 设计规范

详见 `AGENTS.md`，核心要点：

1. **玻璃拟态**：所有面板/卡片/弹窗用半透明背景 + backdrop-blur
2. **Mesh 渐变**：5 层 radial-gradient（紫/蓝/粉/青/橙）
3. **双主题**：所有颜色必须适配 dark/light
4. **大圆角**：面板 24px，卡片 16px，按钮 12px
5. **taskwarrior 日期格式**：YYYYMMDDTHHmmssZ，用 `utils/date.ts` 转换
6. **后端 filter 顺序**：`task status:pending export`（filter 在 export 前）

## v0.2 目标

基于 Figma Make 二次设计后的新设计稿，改造前后端 UI/交互。

### 预期流程

1. 将 `docs/figma-import/` 下的资料发给 Figma Make
2. Figma Make 生成新的设计稿
3. 将新设计稿导出到 `design/v0.2/` 文件夹
4. 基于新设计稿改造前后端代码

### 需要关注的点

- 新设计稿可能改变布局结构、组件样式、交互方式
- 后端 API 可能需要新增接口（如果新设计有新功能）
- 保持 AGENTS.md 中的设计规范
- 保持现有功能不丢失（键盘快捷键、时间追踪、音效等）
